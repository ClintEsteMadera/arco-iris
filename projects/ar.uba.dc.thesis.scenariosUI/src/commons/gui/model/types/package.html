Clases e interfaces para definir "tipos de datos" y su meta-información asociada.

La idea es tener un "diccionario de datos" (implementado por la clase {@link sba.ui.model.types.EditConfigurationManager}) que asocie a un identificador de tipo ({@link sba.ui.model.types.EditType}) cierta meta-información útil para editar y formatear los valores de ese tipo ({@link sba.ui.model.types.EditConfiguration})

Las aplicaciones pueden registrar tipos en el diccionario (generalmente durante la inicialización) de forma tal que la meta-información pueda ser consultada y utilizada por otros componentes del framework.

Ejemplo: 

public static void loadDataTypes() 
{
	//los identificadores se formatean sin separador de miles
	NumberEditParameters id = new NumberEditParameters(10);
	id.grouping=false;
	EditConfigurationManager.getInstance().putNumberConfiguration(Integer.class,ID,id);
		
	//Monto: 8 digitos enteros y 2 decimales
	NumberEditParameters monto = new NumberEditParameters(8, 2);
	EditConfigurationManager.getInstance().putNumberConfiguration(BigDecimal.class,MONTO,monto);

	//Hora de la rueda: 
	DateEditParameters timeStampRueda = new DateEditParameters(DateEditParameters.MODE_HOUR);
	timeStampRueda.showSeconds = true;
	timeStampRueda.setTodayDefaultDate();
	EditConfigurationManager.getInstance().putDateConfiguration(TIMESTAMP_RUEDA,timeStampRueda);
}

En este ejemplo se registran en el diccionario las caracterísitcas de diferentes tipos de datos. Como se puede observar, la información asociada a cada tipo varía según la clase asociada al tipo de dato (Date, Integer, BigDecimal, etc)

Un uso común de los tipos de datos es la asignación de un tipo de dato a una propiedad de un bean.

Ejemplo: 

/**
* Atributos para los modelos de la clase LoteTO
*/
public class LoteTOAttributes extends BeanAttributes 
{
	public static final String ID = "id";
	public static final String MONTO = "monto";
	
	public LoteTOAttributes() 
	{
		super(LoteTO.class);
		
		//setea el tipo de dato para cada propiedad
		setEditConfiguration(MONTO, SubastaTypes.MONTO);
		setEditConfiguration(ID, SubastaTypes.ID);
	}
}

En este ejemplo se asocia a las propiedades de un bean una identificación del tipo de dato correspondiente, mediante la configuración del conjunto de atributos (ver {@link sba.ui.model.bean.BeanAttributes}). De esta forma cada vez que se utilicen estos atributos (ya sea en el despliegue de una colleción de beans en una tabla o de una única instancia en un formulario) la meta-información podrá ser accedia por los componentes visuales para setear máscaras de edición y formateo.
